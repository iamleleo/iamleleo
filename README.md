# 🔬 Engineering Hub
> A personal knowledge system to become a **World-Class Principal Engineer**.

## 🎯 Objective

**Build a deep, intuitive, and understanding knowledge base.**

The diagram below exemplifies what we want to achieve.

```
Bottom-Up                          Top-Down
(Discrete Math)                    (System Design)
     │                                  │
     │    ┌──────────────────────┐      │
     └───▶│  Meet in the Middle  │◀─────┘
          │                      │
          │  Foundry (Capstone)  │
          │  ML Platform Project │
          └──────────────────────┘
```

Traversable in both directions:
- **Why does this work?** (trace down to foundations)
- **What does this enable?** (trace up to applications)

### The Feeling

> "The derivative of an orange is a slice."
Intuition so deep that you don't calculate, you *see and feel*.

### The Goal

> *The goal is not to collect notes. The goal is to become the engineer who doesn't need them!*

## Principles

- **Build to understand**: projects over passive reading
- **Explain to learn**: if you can't teach it, you don't know it
- **Connect ideas**: link notes across areas
- **Question assumptions**: start from first principles
- **Iterate**: notes evolve as understanding deepens

## The Rule
Don't read to prepare. Build until you're stuck, then read to get unstuck.

Just-in-time, not just-in-case.
